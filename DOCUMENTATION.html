OrbitFlow

1. Welcome

OrbitFlow is a premium React SaaS template engineered for modern startups. Built with React 18, TypeScript, Tailwind CSS v4, and Vite, it provides a production-ready foundation for your next web application.

Frontend Template Notice This is a frontend UI template. Authentication, billing logic, forms, and dashboard data are visually simulated for demonstration purposes. You will need to connect your own backend (Node.js, Supabase, Firebase, Laravel, etc.) to make it fully functional.

2. System Requirements

Before you begin, ensure your local development environment meets the following requirements:

  • Node.js: Version 20.0 or higher (LTS recommended).
  • Package Manager: npm (included with Node.js), yarn, or pnpm.
  • Code Editor: Visual Studio Code (recommended) with Prettier and Tailwind CSS IntelliSense extensions.
  • Browser: Modern versions of Chrome, Firefox, Safari, or Edge.

3. Quick Start

Follow these steps to get the template running locally on your machine.

Step 1: Extract the archive

Extract the zip file to your desired local directory.

Step 2: Install dependencies

Open your terminal, navigate to the extracted project folder, and run:

npm install

Step 3: Start the development server

Run the Vite development server with hot-module replacement (HMR):

npm run dev

Your browser should automatically open to http://localhost:5173.

Step 4: Build for Production

When you are ready to deploy, create an optimized production build:

npm run build

This generates a dist/ folder containing all minified static assets.

│   │   └── vertex/         # Marketing sections (Pricing, Features, Testimonials)

│   ├── lib/                # Utility functions (cn for Tailwind class merging)
│   ├── pages/              # Route-level views (Home, Dashboard, Login, etc.)
│   ├── App.tsx             # Main React Router configuration
│   ├── main.tsx            # React application entry point
│   └── style.css           # Global CSS and Tailwind design tokens
├── tailwind.config.js      # Tailwind theme configuration
├── tsconfig.json           # TypeScript configuration
└── vite.config.ts          # Vite bundler configuration

6. Customization Guide

Brand Colors & CSS Variables

OrbitFlow uses Tailwind CSS combined with native CSS variables for easy theming. Open src/style.css to modify the core brand colors:

:root {
  /* Change this to your primary brand color (currently Teal) */
  --orbit-accent-primary: #0ea5a4;
  
  /* Background colors */
  --orbit-base: #090d12;
  --orbit-surface: #0e131f;
}

Changing the Logo

The logo is a standalone component. To replace it with your own SVG or Image:

  1. Open src/components/ui/OrbitLogo.tsx
  2. Replace the existing SVG path inside the component with your own vector data, or replace the entire return statement with an <img src="/your-logo.svg" /> tag.

Updating SEO Metadata

Don't forget to update the global metadata before launching:

  • index.html: Update the <title>, description, and OpenGraph tags.
  • public/robots.txt: Update the sitemap URL domain.
  • public/favicon.ico: Replace with your own favicon.
Placeholder Images The template uses generic placeholder avatars and images to comply with marketplace licensing. You must replace these with your own licensed photography in the src/ directory before publishing.

7. Component Reference

OrbitFlow includes dozens of accessible, highly-polished UI primitives located in src/components/ui/. Most interactive components are built on top of Radix UI primitives.

Button

Supports variant (default, outline, ghost, link, destructive) and size (default, sm, lg, icon). Uses Tailwind merge for safe class overriding.

Dialog / Modal

Accessible modal dialog built with Radix. Use DialogTrigger and DialogContent to create popups without managing z-index state.

DropdownMenu

Supports nested menus, checkboxes, and keyboard navigation. Perfect for user profile menus and table action dropdowns.

Tabs

Accessible tabbed interfaces used extensively in the Settings and Analytics pages.

8. Backend Integration

Google Maps Embed: To use the Google Maps component in production, you must generate your own API key and replace the placeholder in the component code. See Google Maps Documentation.

To turn this frontend template into a working application, you will need to integrate APIs.

Authentication

The auth pages (Login.tsx, Register.tsx) currently use a simulated `setTimeout` to mimic network latency, then write a fake token to localStorage before redirecting to the dashboard.

To connect real auth (e.g., Supabase, Firebase, NextAuth):

  1. Locate the handleSubmit function in the respective auth page.
  2. Remove the `setTimeout` and `localStorage` logic.
  3. Replace it with your authentication provider's login/signup method.
  4. Implement a global Auth Context or state manager (like Zustand/Redux) to protect the /dashboard routes.

Dashboard Data

The charts and tables in the dashboard read from static mock arrays located in src/constants/ and src/data/. Use standard useEffect hooks or a data fetching library like React Query or SWR to fetch JSON from your backend API and populate these components dynamically.

9. Deployment

Because OrbitFlow is a Vite-based Single Page Application (SPA), deployment is fast and inexpensive.

Netlify / Vercel

  1. Push your repository to GitHub, GitLab, or Bitbucket.
  2. Import the project into Netlify or Vercel.
  3. The platform should automatically detect Vite.
  4. Build Command: npm run build
  5. Output Directory: dist

Manual Deployment (cPanel / Static Hosting)

  1. Run npm run build locally.
  2. Upload the contents of the generated dist/ folder to your web server's public_html directory.
  3. Ensure your server is configured to route all 404 requests back to index.html (using an .htaccess fallback for Apache, or try_files for Nginx) to support React Router's client-side navigation.
